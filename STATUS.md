## Human Prompts
Your prompts are the most important part of your work and you should save them
The classical way to use coding assistants is to treat the prompts as ephemeral, type them into a chat field, WhatsApp or Slack messages, or maybe GitHub issues, maybe save a conversation log, and at the same time lovingly tend to the coding artifacts made by the AI in a GitHub repository. I think that is completely backwards: Your prompts (and more generally, whatever your human edits to the codebase) are precisely where you, human, add value and creativity. Given these prompts, any AI agent of the next generation can rebuild your things better than before. They shouldn’t be relegated to some „conversation archive“ deleted after 30 days, not even to non-open, not-trivially-exportable GitHub issues or Slack messages. They should live in the Git repository itself. The parts generated by the agents are fungible and its value will depreciate exponentially.

Therefore, I try not to waste my prompts. Instead, my prompts are either markdown files committed to the repository or empty commit messages: If I am in the main branch (or in a worktree created by my parallel worktree skill), I can effectively apply:`git commit —allow-empty -m „usr: I don’t like your latest edit because X, try Y instead.“.`

Using the STATUS.md pattern (link).

Another advantage of communicating with the bots in committed markdown files is that I can easily give feedback and make scribblings. Then I can do the work:

`codex --dangerously-skip-permissions “work according to these changes by the user: $(echo `git diff HEAD~1 HEAD -- USER_IO.md“ context: $(git log -5), STATUS.md: $(cat STATUS.md) `)`

I can parallelise tasks in a worktree, by using the parallel worktrees skill. In the end, the git commit log and patches of the USER_IO.md become a transcript (with branches) of the conversation.

I haven’t converged on the ultimate way to do it (and it can certainly be automated further), but I am pretty sure that just forgetting the prompt, or ripping it from the context of the artifacts and repos it creates, is the wrong approach. A coding agent can easily distill the „human essence“ of a repository from whatever format one chooses, but it can’t do so if it only the slop part is available.

Obviously one shouldn’t put secrets in there! But one shouldn’t put them into your prompts in the chatboxes either.
